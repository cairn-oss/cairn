# Cairn v0.2.0 — fit the workflow

Cairn moves from a scanner you run into a participant in the change
workflow: it drafts fixes where engineers review changes, and replaces
silent exceptions with documented ones. This release climbs the Trust
Ladder exactly one rung — from *read-only* to *propose*.

## Highlights

- **`cairn propose` (Trust Ladder rung 1).** Renders a scan as a
  review-ready remediation proposal — per-file patches, rationale, and
  security-before-cost sequencing — ready for a pull-request description
  (`gh pr comment --body-file`). It **modifies nothing**; proposals are
  audited under their own action type.
- **Inline suppressions.** `# cairn:ignore RULE reason=...` inside a
  resource grants a documented, per-resource exception. Reasons are
  **mandatory** — a marker without one does not suppress and surfaces as a
  warning. Suppressed findings stay visible in the JSON report and audit
  trail.
- **Three new rules.** SEC010 plaintext HTTP load-balancer listener (the
  port-80 → HTTPS redirect pattern is recognized as correct), REL003
  database deletion protection disabled, COST006 EBS volume attached to
  nothing (with a $/month estimate).

## Upgrade

```bash
pip install --upgrade cairn-iac   # -> 0.2.0
```

No config changes required. JSON output gains additive `suppressed` and
`warnings` fields; `schema_version` stays `1`, so existing consumers are
unaffected. Exit codes are unchanged.

## Compatibility & rollback

Inline markers are inert comments to v0.1.0 and proposals are plain files,
so `pip install cairn-iac==0.1.0` is a clean rollback.

## Verified

169 tests green (≥95% coverage), ruff + `mypy --strict` clean, `propose`
proven byte-non-modifying by a tree-digest test. Full detail in
`CHANGELOG.md`.

## Cloud coverage note

This release is **not yet fully cloud-agnostic**. Its rule packs cover AWS
only.
Terraform for other providers (Azure, GCP, Oracle, VMware
vSphere, ...) is *parsed* but not yet *checked*. Broader multi-cloud and
on-prem coverage arrives in later releases (Azure/GCP/vSphere packs in
v0.4.5; coverage-transparency reporting in v0.5.1, which makes unscanned
resources explicit rather than reporting a misleading "clean"). See
[docs/coverage.md](../docs/coverage.md).
