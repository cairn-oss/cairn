# Cairn v0.4.0 — the knowledge graph

Context and extensibility, kept local. Cairn now understands how your
resources depend on each other, lets the community ship rules, and tells
you when reality has drifted from your code — without a single new network
call or cloud credential.

## Highlights

- **Blast radius on every finding.** A local dependency graph (references,
  `depends_on`, Kubernetes selectors) means a finding now reports what
  transitively depends on the flagged resource. Triage a misconfiguration
  fronting ten downstream resources differently from an isolated one. Shown
  in console/HTML, emitted as `blast_radius` in JSON (schema_version 2).
- **Plugin SDK v1.** Any package advertising the `cairn.rules` entry
  point loads automatically; `cairn rules` shows which rules came from
  which plugin; `--no-plugins` guarantees a builtin-only CI run. A broken
  plugin is reported and skipped, never crashing a scan. The `@rule`
  authoring surface is now frozen as public API.
- **Drift detection.** `cairn drift ./infra state.json` compares your
  declared Terraform against a `terraform show -json` snapshot **you**
  produce — Cairn reads no cloud credentials — classifying every
  resource as in-sync, missing, or unmanaged. Exits 1 on drift.

## Upgrade

```bash
pip install --upgrade cairn-iac   # -> 0.4.0
```

JSON consumers: `schema_version` is now `2` (only additive fields). `pip
install cairn-iac==0.3.0` is a clean rollback.
