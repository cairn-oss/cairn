# Cairn v0.4.5 — multi-cloud & on-prem

Cairn is now cloud-agnostic in fact, not just in architecture. Scan
AWS, Azure, GCP, VMware vSphere, and Kubernetes through one pipeline, with
a provider dimension you can filter and report on — and run the whole thing
air-gapped.

## Highlights

- **Azure, GCP, and vSphere rule packs** join AWS and Kubernetes: 15 new
  rules covering network exposure, public storage/databases, encryption,
  oversizing (with cost estimates for Azure/GCP), and tagging/ownership.
  On-prem vSphere focuses on reliability, thin provisioning, and ownership.
- **Provider dimension.** Every finding is tagged with its provider,
  inferred from the resource type and emitted in JSON. Filter with
  `cairn scan --provider aws,azure` (or `vsphere` for on-prem only).
- **Air-gapped operation.** The whole tool already runs with no internet
  access; `docs/air-gapped.md` covers offline install (wheel or container)
  and how to verify zero egress.

## The honest on-prem boundary

On-prem hardware has no per-hour cloud pricing, so cost findings are
cloud-only. Security, reliability, and governance apply everywhere — which
is where on-prem value concentrates. Cairn says so rather than inventing
dollar figures.

## Upgrade

```bash
pip install --upgrade cairn-iac   # -> 0.4.5
```

Purely additive: existing AWS/Kubernetes behavior is unchanged; new
findings appear only for Azure/GCP/vSphere resources. `pip install
cairn-iac==0.4.0` is a clean rollback.
