# Cairn v0.5.0 — guarded autonomy

Cairn takes its first, carefully-fenced step from *proposing* changes to
*applying* them — Trust Ladder rung 2 — for a provably-safe class of fixes,
under four independent guards.

## Highlights

- **`cairn fix`** applies a hard-coded whitelist of non-destructive,
  attribute-only fixes: gp2→gp3, enable RDS/EBS encryption, deletion
  protection. Dry-run by default; `--apply` writes.
- **Four guards stack.** The eligible-fix whitelist lives in code (a
  scanned repo can't extend it); each class must be granted in policy
  (`autonomy.allow`, with `[]` as a kill switch); applying refuses on a
  dirty or non-git worktree so every change is a reviewable, revertible
  diff; and each applied change is audited with before/after hashes.
- **Never guesses.** A fix whose exact expected text isn't found is skipped
  and reported — a wrong edit to infra code is worse than no edit.

## Upgrade

```bash
pip install --upgrade cairn-iac   # -> 0.5.0
```

`fix` is opt-in and dry-run by default — upgrading changes nothing until
you grant categories and pass `--apply`. Applied changes are ordinary git
history; `pip install cairn-iac==0.4.5` restores propose-only behavior.
