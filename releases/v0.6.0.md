# Cairn v0.6.0 — first public release

Cairn's first public release on PyPI. Cairn is a local-first
Infrastructure-as-Code auditor: it reads cost, security, reliability and
governance in a single pass over your Terraform, reconciles the trade-offs
when findings collide on the same resource, and writes the fix — the exact
line to change, not a vague "misconfiguration detected."

## Highlights

- **`pip install cairn-iac`**, then `cairn scan ./infra`. No account, no
  upload, zero network calls — your infrastructure never leaves your machine.
- **One pass, four lenses** — 42 rules across AWS, Azure, GCP, Kubernetes and
  on-prem vSphere, with dollar estimates sitting next to cost findings.
- **Trade-off reconciliation** — when a cost fix and a security fix touch the
  same resource, Cairn tells you which to do first (sequence the security fix,
  then right-size — resizing an exposed resource first just makes the breach
  cheaper to run).
- **Honest coverage** — a scan is only reported "clean" when every resource
  belongs to a covered provider; otherwise it says "checked N of M" and lists
  what it did not scan. No fake green.
- **Guarded autonomy** — `cairn fix --apply` writes only a hard-coded whitelist
  of non-destructive changes, only on a clean git worktree, dry-run by default,
  with before/after audit hashes.

## Configuration & CLI

- Config file: `.cairn.yaml` (or `--config`).
- Inline suppressions: `# cairn:ignore RULE reason=...` (reason mandatory).
- Commands: `scan`, `fix`, `propose`, `diff`, `drift`, `providers`, `license`.
- Output formats: console, JSON, SARIF (GitHub code scanning), HTML, Markdown.

## Supply chain

Signed, reproducible releases via PyPI Trusted Publishing (OIDC), with SLSA
build-provenance attestation and a CycloneDX SBOM attached to each release.

## Install

```
pip install cairn-iac
cairn scan ./infra
```
